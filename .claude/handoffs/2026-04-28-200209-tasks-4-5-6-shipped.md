**Key design pinned by 4 rounds of review:**
- gnhf has NO machine-readable stdout (TUI). Outcome lives in `<cwd>/.gnhf/runs/<runId>/gnhf.log` (JSONL).
- gnhf `--worktree` is a boolean flag, NOT a path → use `cmd.Dir`.
- Real Status enum is `{stopped, aborted}`, with `Reason` derived from `lastMessage` substrings.
- `notes.md` lives at `.gnhf/runs/<runId>/notes.md` (per-run).
- runId discovery uses **set difference on directory NAMES** (NOT mtime) — robust against pre-existing dirs that get touched post-spawn.
- Graceful cancel uses `Setpgid` + signals process group → grace → SIGKILL. Default `exec.CommandContext` would lose the final `run:complete` line.
- Cwd-resolved exclude path uses `git rev-parse --absolute-git-dir --git-common-dir` (single fork, no fallback).

**Key design pinned by 3 Codex rounds:**
- Recovery paths must distinguish "outbox marker exists" from "post happened". `OutboxCheck` returns `lark_comment_id` — empty means we haven't posted, non-empty means we did. Implementer dispatch is the FIRST flow to call `OutboxMarkPosted` on success (existing dispatchAnswer/Compact/Normal don't — pre-existing inconsistency, not retrofitted).
- Pre-spawn idempotency guard via `GetImplementerRunByCommentID`: case A (no row → fresh), case B (row finalized → recover post), case C (row not finalized → finalize as failed + post + mark processed).
- DB lookup error → `MarkDeferred(now+60s)` instead of fall-through to fresh spawn (prevents duplicate gnhf runs on transient DB hiccups).
- Outer flock held through dispatch; DispatchImplement assumes caller holds it (godoc'd LOCK CONTRACT).

## Carry-over risks (open, deferred to M3)

1. **`FinalizeImplementerRun` failure swallowed** — TODO(M3) marker in `dispatch.go` near the FinalizeImplementerRun error log. Run telemetry permanently lost on DB write failure mid-finalize.
2. **`started_at` unit mismatch** — `Unix()` (seconds) vs `finished_at` `UnixMilli()`. Inherited from migration0004 (Task 1). Mixed units make duration queries awkward. Fix would need migration0006 to align both to UnixMilli.
3. **Token cap (M3)** — Per-task token capping not implemented. Hard bounds via MaxIterations=30 + Timeout=4h sufficient for MVP under Claude Max. Revisit if: (a) move off Max to pay-per-token API, (b) per-task fairness becomes a constraint, (c) operator wants budget visibility.
4. **No flush worker** — Implementer dispatch posts inline (mirrors dispatchNormal). Lark API outage during 19:00–05:00 window blocks dispatcher. Acceptable for MVP — row marked processed only after post attempt, replay fires if MarkInboxProcessed never lands.
5. **Implementer dispatch is FIRST flow to call `OutboxMarkPosted`** — existing dispatchAnswer/Compact/Normal don't. Inconsistent. Not retrofitted in this branch (out of scope).
